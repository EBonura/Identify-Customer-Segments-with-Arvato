    """
    Perform feature trimming, re-encoding, and engineering for demographics
    data

    INPUT: Demographics DataFrame
    OUTPUT: Trimmed and cleaned demographics DataFrame
    """
    print ('1) convert missing value codes into NaNs')
    
    # Removing the additional value to avoid conflicts with the imputer, it won't affect the 'customer' dataset anyway
    df = df[df['GEBAEUDETYP'] != 5.0]
    
    def try_convert(x): # Helper function
        try:
            return int(x)
        except:
            return x

    df_processed = pd.DataFrame()
    for col in df.columns:
        value_list = feat_info.loc[col]['missing_or_unknown'].strip('][').split(',')
        df_processed[col] = df[col].replace([try_convert(x) for x in value_list], np.nan)

    # remove selected columns and rows, ...
    print('2a) get how many missing values for each col')
    df_processed_count = df_processed.isnull().sum()
    df_processed_missing = (df_processed_count[df_processed_count > 0] / df_processed.shape[0]*100).sort_values(ascending=True)

    print('2b) Establish a threshold and keep only the values above that threshold')
    col_to_rmv = [x for x in df_processed_missing[df_processed_missing > threshold * 100].index]

    df_clean = pd.DataFrame()
    for col in df_processed.columns:
        if col not in col_to_rmv:
            df_clean[col] = df_processed[col]

    # select, re-encode, and engineer column values.
    print('3a) Get feat_info but only with the columns I kept in the original dataframe')
    feat_info_clean = feat_info.copy()

    for column in feat_info.index.tolist():
        if column not in df_clean.columns:
            print(column, 'dropped')
            feat_info_clean.drop([column], inplace = True)

    print('3b) Fill all the nan with the most common column value, this to avoid issues in the One-Hot encoding')
    cat_col = feat_info_clean[feat_info_clean.type == 'categorical'].index.tolist()
    df_clean[cat_col] = df_clean[cat_col].fillna(df_clean[cat_col].mode().iloc[0])

    print('3c) Re-encode categorical variable(s) to be kept in the analysis')
    df_clean['OST_WEST_KZ'] = df_clean['OST_WEST_KZ'].map({'W':0, 'O':1})
    one_hot_col = [col for col in cat_col if len(df_clean[col].unique())>2]
    df_clean_onehot = pd.get_dummies(df_clean, columns=one_hot_col)

    decade_map = {1:'40s', 2:'40s', 3:'50s', 4:'50s', 5:'60s', 6:'60s', 7:'60s', 8:'70s', 9:'70s', 10:'80s', 11:'80s', 12:'80s', 13:'80s', 14:'90s', 15:'90s'}
    movement_map = {1:0, 2:1, 3:0, 4:1, 5:0, 6:1, 7:1, 8:0, 9:1, 10:0, 11:1, 12:0, 13:1, 14:0, 15:1}
    df_clean_onehot['JUGENDJAHRE_DECADE'] = df_clean_onehot['PRAEGENDE_JUGENDJAHRE'].map(decade_map)
    df_clean_onehot['JUGENDJAHRE_MOVEMENT'] = df_clean_onehot['PRAEGENDE_JUGENDJAHRE'].map(movement_map)

    df_clean_onehot['WEALTH'] = df_clean_onehot['CAMEO_INTL_2015'].apply(lambda x: str(x)[0] if not pd.isnull(x) else x)
    df_clean_onehot['LIFESTAGE'] = df_clean_onehot['CAMEO_INTL_2015'].apply(lambda x: str(x)[1] if not pd.isnull(x) else x)

    one_hot_col2 = ['JUGENDJAHRE_DECADE', 'JUGENDJAHRE_MOVEMENT', 'WEALTH','LIFESTAGE']
    df_clean_onehot2 = pd.get_dummies(df_clean_onehot, columns=one_hot_col2)

    print('3d) Drop columns I don\'t need')
    df_clean_onehot2.drop(['PRAEGENDE_JUGENDJAHRE','CAMEO_INTL_2015'], axis=1, inplace=True)

    # Return the cleaned dataframe.
    return df_clean_onehot2

Step 2.3: Interpret Principal Components¶

Now that we have our transformed principal components, it's a nice idea to check out the weight of each variable on the first few components to see if they can be interpreted in some fashion.

As a reminder, each principal component is a unit vector that points in the direction of highest variance (after accounting for the variance captured by earlier principal components). The further a weight is from zero, the more the principal component is in the direction of the corresponding feature. If two features have large weights of the same sign (both positive or both negative), then increases in one tend expect to be associated with increases in the other. To contrast, features with different signs can be expected to show a negative correlation: increases in one variable should result in a decrease in the other.

  • To investigate the features, you should map each weight to their corresponding feature name, then sort the features according to weight. The most interesting features for each principal component, then, will be those at the beginning and end of the sorted list. Use the data dictionary document to help you understand these most prominent features, their relationships, and what a positive or negative value on the principal component might indicate.
  • You should investigate and interpret feature associations from the first three principal components in this substep. To help facilitate this, you should write a function that you can call at any time to print the sorted list of feature weights, for the i-th principal component. This might come in handy in the next step of the project, when you interpret the tendencies of the discovered clusters.

Discussion 3.3: Compare Customer Data to Demographics Data¶

By isolating the differences I was able to pinpoint the profile of the customers popular with the company:

  • Cluster 1 and 4 are overrepresented = popular with the company:

    • FINANZ features (describing the finance topology) are higher
    • HH_EINKOMMEN_SCORE = 2 means very high income
    • LP_LEBENSPHASE_GROB = 12: high-income earners of higher age from multiperson households
    • LP_LEBENSPHASE_FEIN = 40: top earners at retirement age from multiperson households
    • SEMIO values: lower affinity

  • Clusters 5 and 8 are underrepresented = unpopular with the company:

    • FINANZ features are lower
    • LP_LEBENSPHASE_GROB = 1: single low-income and average earners of younger age
    • SEMIO values: higher affinity
    • JUGENDJAHRE_DECADE_90s = 1 + JUGENDJAHRE_DECADE_50s = 0: younger people

Although some features are mixed, seems that Older, middle-aged and elderly traditional people, house owners, with average-high income are the best target for the company as they more represent demographics popular with them.
